# TRIP 방법론·수식 정리 (이해용)

주 방법론 논문: **Espadaler-Clapés 외 (2025), TRIP 32, 101490** (맨체스터 2곳, 송도와 같은 Fonod팀·파이프라인). 정독은 상충 계산(06) 직전에 하고, 아래는 개념 수준 정리.

## 방법론 한 줄
드론 궤적에서 **가까워지는 차량 쌍**을 찾아, **TTC·PET**로 "얼마나 아슬아슬했나"를 재서 상충으로 센다. TRIP은 이걸로 안전·혼잡 **모니터링**을 함.

## TTC (추종 상충) — 얼마 후 충돌하나
```
TTC = dist(sub, surr) / (v_sub − v_surr),   단 v_sub > v_surr
```
| 변수 | 의미 |
|---|---|
| sub | 대상 차 = 뒤차(따라가는 차) |
| surr | 주변 차 = 앞차(선행차) |
| dist(sub,surr) | 두 차 사이 거리(m) |
| v_sub, v_surr | 각 차 속도(우리가 재계산) |
| v_sub>v_surr | 뒤차가 더 빨라 거리가 줄 때만(가까워져야 위험) |

의미: 지금 속도·거리 그대로면 **몇 초 후 충돌**하나. 작을수록 위험.

## PET (교차 상충) — 얼마나 아슬하게 비껴갔나
```
PET = t²c − t¹c
```
| 변수 | 의미 |
|---|---|
| 공통 지점 | 두 차 경로가 만나는 점 |
| t¹c | 첫 차가 그 점 통과한 시각 |
| t²c | 둘째 차가 그 점 통과한 시각 |

의미: 한 차가 지난 뒤 다른 차가 오기까지 **시간 간격**. 실제 충돌은 없었지만 거의 동시에 지났으면(작을수록) 아슬아슬.

## 추종 vs 교차 — 왜 나누나
- 추종: 두 차 같은 방향, 앞뒤 → **TTC** (뒤차가 앞차 따라잡는 위험)
- 교차: 두 차 경로가 각을 이루며 만남(직진 vs 좌회전 등) → **PET** (같은 점을 시차 두고 통과)
- 왜: 우리 데이터에 **신호현시 정보가 없어서**. TTC는 추종에, PET는 교차에 자연스럽게 맞음.

## 방위각(azimuth) — 방향으로 둘을 가름
- 각 차의 진행 방향 각도(속도벡터에서 atan2). 나침반 방향 같은 것.
- **두 차 방위각 차이 ≥ 30°면 교차, 작으면 추종** (TRIP이 30° 사용).

## 정독(06 직전)에서 확정할 것
30°의 정확한 적용, 평활 방식, 거리·통과지점 정의 세부, 우리 조정(속도 재계산·PET 세그먼트 한정·이동류 OD·치수 민감도) 반영 방법.
